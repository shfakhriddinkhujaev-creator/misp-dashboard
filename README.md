# ҲИБКК - Региональный экономический дашборд (прототип)

Прототип аналитической системы для мониторинга экономического здоровья регионов
Узбекистана, реализующий концепцию композитного индекса **ҲИБКК**
(Ҳудудий иқтисодий барқарорлик композит кўрсаткичи).

## Что внутри

- **Executive Summary** - национальный балл, топ-критические области, лучший
  рост по сигналам, активные предупреждения.
- **Карта 14 областей** с цветовым кодированием по 4-уровневой шкале сигналов
  (Яхши / Ўртача / Ёмон / Хавфли) и тултипом по hover.
- **Сетка drill-down** - 14 кликабельных карточек регионов под картой
  (отсортированы по рангу), один клик → профиль области.
- **Профиль области:** hero-header, KPI 7 блоков, радар-диаграмма (текущий vs
  предыдущий квартал), 12-месячный тренд, highlight-карта с подсвеченным
  регионом, drill-down на туманы (treemap + таблица), активные сигналы,
  похожие области, авто-генерация политических рекомендаций.
- **Sidebar** с фильтрами (квартал/год) и навигацией по регионам.
- **Эрта огоҳлантириш тизими** - система раннего предупреждения по 8 прокси
  с 4-уровневой шкалой реакции.

## Стек

- **Streamlit 1.32+** - реактивный UI, без отдельного фронтенда.
- **Plotly Express** - все визуализации (choropleth_map, radar, line, bar, treemap).
- **pandas + numpy** - данные.

## Запуск локально

```bash
# 1. Клонировать репо
git clone https://github.com/shfakhriddinkhujaev-creator/misp-dashboard.git
cd misp-dashboard

# 2. Виртуальное окружение
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Запустить
```

## Архитектурные решения

**Карта через `px.choropleth_map` (MapLibre).** GeoJSON областей берётся из
[akbartus/GeoJSON-Uzbekistan](https://github.com/akbartus/GeoJSON-Uzbekistan)
(GPL-3.0, OSM-derived) - единственный публичный источник с актуальными
границами всех 14 областей. Файл скачивается при первом запуске с локальным
кэшем как fallback на случай отсутствия сети.

**Schema-agnostic matcher региональных имён.** В `make_plotly_choropleth`
имена областей резолвятся через каскад: прямой lookup в
`GEO_NAME_MAP` → нормализация (lowercase, удаление диакритики/пробелов) →
двусторонний substring matching. Устойчиво к транслитерационным вариантам
(Sirdarya↔Syrdarya, Khorezm↔Xorazm, Bukhara↔Buxoro и т.д.).

**Нормализация GeoJSON.** Akbartus поставляет геометрию завёрнутой в
`GeometryCollection`, что Plotly рендерит как пустоту без ошибки.
`normalize_geojson_for_plotly` рекурсивно flatten'ит любую геометрию в
`MultiPolygon` и проставляет `feature.id` из `ADM1_UZ` для прямого матчинга
с `featureidkey="id"`.

**Drill-down - два пути:**
1. **Сетка карточек регионов** под картой (7×2): emoji-сигнал, имя, балл, ранг.
   Клик → `st.session_state.selected_region` → ререндер показывает профиль.
2. **Sidebar selectbox** - текстовая навигация для пользователей,
   которые предпочитают список.

В прототипе native-клик по choropleth не используется намеренно: в
комбинации Streamlit 1.32+ / Plotly 6 / streamlit-folium 0.27 click-event
pipeline для choropleth-слоёв оказался ненадёжным (известная проблема,
[streamlit-folium#192](https://github.com/randyzwitch/streamlit-folium/issues/192)
и аналогично для `st.plotly_chart(on_select="rerun")` на MapLibre-картах).
Карточки + sidebar дают тот же UX без зависимости от click-handling библиотек,
плюс mobile-friendly и accessibility-ready.

**Расчёт районов.** Для уровня туманов (~199 синтетических, цель 215+)
используется treemap, потому что публичные ADM2-границы доступны лишь для
~5 областей. В продакшен-версии после получения ADM2 от ЎзСТАТ/OCHA логика
drill-down остаётся та же - меняется только источник GeoJSON.

**Данные синтетические, методология реальная.** Базовые баллы откалиброваны
по мокапу (Тошкент ш. = 74, ҚҚР = 28 и т.д.). Композитный индекс
пересчитывается из блочных баллов с весами 22/18/16/15/12/10/7%, что
обеспечивает математическую согласованность с концепцией.

## Roadmap

- **Реальные ADM2-границы** для всех 14 областей (OCHA HDX / ЎзСТАТ).
- **Экспорт PDF** профиля области через ReportLab.
- **Сравнительный режим** - 2-3 области одновременно в радаре.
- **ML-прогноз** ҲИБКК на 3-6 месяцев (Prophet) на исторических данных.
- **API endpoint** для внешних исследователей (открытые данные).
- **Аутентификация и роли** (общественный / областной аналитик / эксперт).
- **Anomaly detection** на блочных индикаторах.
